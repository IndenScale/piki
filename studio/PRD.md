## 1. 产品定位

### 1.1 一句话定义

**Piki Studio 是面向工程设计工程师的浏览器端可视化 IDE**，让工程师在提交设计前快速预览三维布局、检查空间冲突、并直接回到文本修正。

### 1.2 与 piki CLI 的关系

| | piki CLI | Piki Studio |
|---|---|---|
| **用户** | 运维工程师、CI/CD 流水线 | 设计工程师、现场工程师 |
| **场景** | 批量检查、自动化报告、Git 集成 | 可视化验证、交互式排查、设计审查 |
| **交互方式** | 命令行 + 文本编辑器 | 浏览器 + 3D 视口 + 面板 |
| **成功标准** | 检查覆盖率、规则执行速度、报告质量 | 首屏加载时间、交互延迟、视觉保真 |

**设计原则**：Studio 是 CLI 的**可视化补充**，不是替代。设计的源头仍然是文本，Studio 让"看效果"更直观。

### 1.3 目标用户画像

- **初级设计工程师**：刚接触 piki，需要可视化辅助理解 YAML 与 3D 空间的对应关系
- **资深设计工程师**：快速审查大型项目，定位空间冲突的具体位置
- **现场工程师**：在施工现场用平板查看设计，确认安装顺序和空间关系

## 3. 非功能需求

### 3.1 性能

| 指标 | 目标 | 说明 |
|---|---|---|
| 首屏加载 | < 3s（本地） | 不含 WASM，纯文本解析 |
| 场景渲染 | > 30 FPS | 1000 个 Box 对象 |
| 项目加载 | < 5s | 1000 个 YAML 实例文件 |
| 内存占用 | < 200MB | 典型项目 |

### 3.2 兼容性

| 环境 | 支持级别 |
|---|---|
| Chrome / Edge | 完全支持 |
| Firefox | 完全支持 |
| Safari | 有限支持（File System Access API 限制） |
| 移动端浏览器 | 暂不保证 |

### 3.3 可访问性

- 所有交互元素支持键盘导航
- 颜色对比度符合 WCAG 2.1 AA 标准
- 3D 视口提供文字替代描述（选中对象时朗读名称和类型）

## 4. 设计原则

1. **Text-Native First**：任何在 Studio 中可做的操作，都必须在文本中可表达。Studio 是文本的视图，不是编辑器。
2. **即时反馈**：用户操作（选择、缩放、切换面板）必须在 100ms 内有视觉响应。
3. **渐进增强**：基础功能不依赖 WASM，复杂渲染（材质、动画）作为可选增强。
4. **领域无关**：核心 UI 不硬编码任何领域的概念（如"机柜""PDU"），通过插件扩展。

## 6. 风险与假设

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| File System Access API 被限制 | 高 | 提供 ZIP 上传作为降级方案 |
| USDA 解析器无法覆盖复杂场景 | 中 | 保留 USD WASM 集成路径 |
| Three.js 性能瓶颈（>5000 对象） | 中 | 引入 LOD、实例化渲染、八叉树剔除 |
| 属性编辑回写导致 YAML 格式破坏 | 高 | 使用 AST 级编辑，保留注释和格式 |
