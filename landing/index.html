piki 如何工作

三条命令，把你的设计变成可版本控制、可自动检查、可多人协作的代码。

1

声明

用 YAML 描述设备、机柜、PDU、线缆等工程对象。结构清晰，LLM 可直接读写。

device:
  id: srv-01
  family: server
  rack: rack-A
  u_position: 3
2

检查

用 Python 函数表达业务规则。piki 在每次 commit 前自动运行检查，输出精确诊断。

@rule("POWER-001", "PDU 功率预算")
def check_pdu_budget(ctx):
    ...
3

生成

从同一份 YAML 生成 BOM、功率预算、线缆清单、机柜面板图，甚至 3D 预览。

piki generate bom-csv
piki generate power-budget
piki generate rack-face-panel

核心能力

从 Schema 校验到 3D 碰撞检测，覆盖工程设计验证的全链路。

Schema 校验

字段类型、范围、必填项自动检查，错误精确到行

规则引擎

@rule 装饰器注册任意业务规则，pytest 风格断言

Instance / Layout 分离

设备身份与部署决策独立管理，方案比选用 Git 分支

交付物生成

BOM CSV · 机柜面板图 · 功率预算 · 线缆清单

AABB 碰撞检测

同一机柜内设备 3D 空间冲突检测

Piki Studio

浏览器端 3D 预览 IDE，打开项目即看布局

Git + CI/CD 原生

pre-commit hook + GitHub Actions 模板

LSP 诊断格式

human / json / junit / markdown，IDE 可直接消费

快速开始

三条命令，跑通你的第一个 piki 项目。

terminal 
# 安装（Python 3.11+）
pip install piki

# 初始化项目
piki init --plugin telecom

# 运行设计检查
piki check

# 生成报告
piki report --format markdown

# 导出交付物
piki generate bom-csv
