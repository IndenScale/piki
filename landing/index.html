piki 是面向工程设计的文本原生声明式系统建模框架。用 YAML 声明工程对象，用 Python 规则检查设计合理性，用 Git 管理设计演进。

rules/power.py 
@rule("TELECOM-POWER-001", "PDU 功率预算检查")
def check_pdu_budget(ctx):
    for pdu in ctx.query("pdus"):
        devices = ctx.query("devices", pdu_id=pdu.id)
        load = sum(d.resolved.tdp_w for d in devices)
        ratio = load / pdu.resolved.capacity_w
        assert ratio <= 0.8, \
            f"{pdu.id} 负载率 {ratio:.1%}，超过 80% 阈值"

你不需要记住检查功率预算。规则会记住。

工程设计为什么需要 piki？

传统工具把设计意图锁在图形界面和专有格式里，协作、验证和自动化都变得异常困难。

规则靠人脑记

功率预算、线缆长度、安全间距……资深工程师的经验无法被系统执行，新人更容易出错。

图纸难以 diff

二进制 CAD/BIM 文件让 Git 无能为力，版本回退、代码评审、分支合并都无从谈起。

自动化成本高

想要 CI 自动检查设计？先要解析专有格式、适配复杂 API，大部分团队望而却步。

piki 如何工作

三条命令，把你的设计变成可版本控制、可自动检查、可多人协作的代码。

1

声明

用 YAML 描述设备、机柜、PDU、线缆等工程对象。结构清晰，LLM 可直接读写。

device:
  id: srv-01
  family: server
  rack: rack-A
  u_position: 3
2

检查

用 Python 函数表达业务规则。piki 在每次 commit 前自动运行检查，并输出精确诊断。

@rule("POWER-001", "PDU 功率预算")
def check_pdu_budget(ctx):
    ...
3

生成

从同一份 YAML 生成 BOM、功率预算、线缆清单、机柜面板图，甚至 3D 预览。

piki generate bom-csv
piki generate power-budget
piki generate rack-face-panel

快速开始

三条命令，跑通你的第一个 piki 项目。

terminal 
# 安装 piki（Python 3.11+）
pip install piki

# 初始化项目，选择行业插件
piki init --plugin telecom

# 运行设计检查
piki check

# 生成 Markdown 报告
piki report --format markdown

# 导出 BOM 清单
piki generate bom-csv
